# Structured Content Components

## Quiz Renderer

`QuizRenderer` displays interactive quizzes generated by the quiz agent.

### Quiz JSON Format

```json
{
  "title": "Quiz Title",
  "questions": [
    {
      "id": 1,
      "type": "multiple_choice",
      "question": "What is...?",
      "options": ["A", "B", "C", "D"],
      "correct_answer": "B",
      "explanation": "Because..."
    },
    {
      "id": 2,
      "type": "true_false",
      "question": "Is X true?",
      "correct_answer": "True",
      "explanation": "..."
    },
    {
      "id": 3,
      "type": "short_answer",
      "question": "Explain...",
      "correct_answer": "Key concept",
      "explanation": "..."
    }
  ]
}
```

### State Persistence

Quiz state (selected answers, revealed explanations, short answer text) is stored in the message's `metadata.quizState` field. When the user interacts with a question, the state is PATCHed to `/api/chat/messages/{id}` so answers survive page reloads.

State shape per question:
```typescript
{selected: string | null, revealed: boolean, shortAnswer: string}
```

## Chart Renderer

`ChartRenderer` dispatches to specialized visualization components based on the JSON `type` field.

### Supported Types

| Type | Component | Description |
|------|-----------|-------------|
| `mermaid` | `MermaidDiagram` | Flowcharts, sequence diagrams, mindmaps, gantt charts |
| `bar`, `pie` | Built-in renderers | Simple chart visualizations |
| `line` | `LineChart` | Multi-series line charts |
| `radar` | `RadarChart` | Radar/spider charts |
| `comparison_table` | `ComparisonTable` | Side-by-side data comparison tables |

### Chart JSON Format

```json
{
  "type": "mermaid",
  "title": "System Architecture",
  "code": "graph LR\n  A-->B\n  B-->C"
}
```

## Mermaid Diagram Rendering

`visualizations/MermaidDiagram.tsx` handles mermaid diagram rendering with several defenses against common issues.

### Singleton Initialization

Mermaid is initialized once via `getMermaid()`. Re-initializing mermaid crashes the library, so a module-level promise ensures `mermaid.initialize()` is called exactly once.

### Serial Render Queue

Mermaid's `render()` is not concurrency-safe — calling it simultaneously from multiple components produces corrupt SVGs. A module-level `renderQueue` (promise chain) serializes all render calls:

```
queueRender() → waits for previous render → calls mermaid.render() → resolves
```

### Theme Configuration

The mermaid theme uses zinc-scale colors matching the dark chat UI:

- Node backgrounds: `#27272a`
- Text: `#e4e4e7`
- Borders: `#3f3f46`
- Lines: `#52525b`
- Background: transparent

### Security

`securityLevel: "strict"` prevents DOM injection through mermaid diagram definitions. This is important because diagram code is LLM-generated and could contain malicious content.

### Mindmap Fix

`fixMindmapIndentation()` repairs flat mindmap output that LLMs commonly produce. LLMs tend to output all nodes at the same indentation level; this function reconstructs the tree structure.

### Pan and Zoom

Large diagrams are wrapped in `react-zoom-pan-pinch` (`TransformWrapper` + `TransformComponent`). On render completion, the diagram auto-fits to the container width with a reset button to restore the default view.

## Thinking Block

`ThinkingBlock` displays the reasoning steps and tool call activity that happens before the assistant's response appears.

Entries are either:
- **Text** — reasoning steps like "Searching the web for..." or "Found 5 results"
- **Tool call** — tool name with status indicator (running spinner, done checkmark, error icon)

During streaming, the block is expanded by default. After streaming completes, it collapses to a summary line showing the count and total duration. The user can expand it to review the steps.

## Markdown Rendering

Assistant text is rendered with [Streamdown](https://github.com/nichochar/streamdown), a streaming-optimized markdown renderer. Configuration:

- **Syntax highlighting** via Shiki (themes: `github-light`, `github-dark`)
- **Remediation** enabled for links, images, bold, italic, inline code, and KaTeX math
- Streaming-safe: handles incomplete markdown during token-by-token delivery
