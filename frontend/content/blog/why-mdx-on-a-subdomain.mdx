The simplest way to ship a blog is also usually the worst: a new repo, a new
deploy, a duplicated design system, and two places to fix every bug.

We took the other route. `blog.runaxai.com` is served by the exact same
Next.js app as `runaxai.com`. One build, one deploy, one design system.

## The routing trick

A single `middleware.ts` inspects the incoming `Host` header. If the request
is for `blog.runaxai.com`, it rewrites the URL to `/blog/…` internally. The
URL bar stays clean (`blog.runaxai.com/some-post`), but the route lives at
`app/blog/[slug]`.

```ts
const host = request.headers.get("host") ?? "";
if (host.startsWith("blog.")) {
  return NextResponse.rewrite(new URL(`/blog${pathname}`, request.url));
}
```

## Cloudflare tunnel + Kubernetes

The cluster doesn't have a public IP — a `cloudflared` deployment opens an
outbound tunnel to Cloudflare's edge. To add a new subdomain you map a
"Public Hostname" in the Zero Trust dashboard from `blog.runaxai.com` to the
in-cluster frontend service. Cloudflare auto-creates the CNAME. Then we add
the host to the Traefik Ingress so it accepts the new `Host` header.

No new pods. No new certs. Same code path.
